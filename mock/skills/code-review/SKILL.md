# Code Review

When asked to review code, follow these steps:

1. Check for correctness — does the code do what it claims?
2. Check for edge cases — are boundary conditions handled?
3. Check for style — does it follow project conventions?
4. Check for security — are there injection risks, hardcoded secrets, or unsafe operations?
5. Suggest improvements — but only if they meaningfully improve the code

Format your review as a list of findings with severity (critical / warning / suggestion).
